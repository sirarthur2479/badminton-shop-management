PIN Login

Open the staff panel at /#/staff. You'll see a numeric keypad — enter the 4-digit PIN to get in.

Save it to your phone Add /#/staff to your phone's home screen (iOS: Share → Add to Home Screen; Android: Chrome menu → Add to Home Screen). It opens straight to the PIN screen — no URL to type each time.

The session stays unlocked until you tap Logout at the bottom of the dashboard, or close the browser. On a shared device, log out at the end of your shift.

Daily Routine

A suggested order of operations for each working day:

Suggested daily flow
Opening
Check the Inquiries tile — if there's a red badge, open Inquiries and handle new ones before the queue gets busy.
Morning
Open the Order Queue. Any Pending orders should be moved to In Progress as you start them. Keep the queue status accurate — it's how the owner tracks what's being worked on remotely.
Completion
Mark orders as Done the moment you finish restringing. This triggers the customer's "ready for pickup" email automatically.
Pickup
Mark as Picked Up when the customer collects. This clears the job from the active queue.
End of day
Scan the queue for anything stuck in In Progress — if a job is stalled, move it back to Pending so it doesn't get lost overnight.

Working the Queue

Every stringing order submitted from the kiosk appears in the Order Queue. The queue is sorted newest-first by default.

Finding an order

Scan the queue visually — each row shows the customer name and racket/string/tension at a glance. If the queue is long, look for the customer's name in the Name column. There's no search at this stage — scroll through the list.

Expanding an order

Click anywhere on a row (except the status badge) to expand it. The expanded view shows:

FieldWhat to use it for
Racket brand & modelIdentify the frame — match with the physical racket before you start
String brand & modelPull the right string from stock
Tension (lbs)Set your stringing machine to this value
NotesCustomer's special instructions — read before you start
Phone / EmailContact the customer if there's a problem with the racket
Always read the Notes field Customers use Notes for things like "frame cracked near grommet 3" or "use the lighter weight version of this string". These affect how — or whether — you proceed.

When something is wrong with the racket

If you find a damaged grommet, a cracked frame, or anything that prevents stringing: contact the customer using the phone/email in the expanded row, then move the order back to Pending until resolved. Don't mark it Done.

Updating Status

Tap the coloured status badge on any order row to cycle it forward:

FromTap toWhen
PendingIn ProgressWhen you pick up the racket and start stringing
In ProgressDoneImmediately when you finish — this emails the customer
DonePicked UpWhen the customer collects the racket in person
Done sends an email The moment you mark an order Done, an email goes to the customer automatically (if they provided an email address when registering). Mark it Done only when the racket is actually finished and waiting on the shelf.

Correcting a status

If you advance a status by mistake — for example, marking Done when it isn't — the status cycles in one direction only. To reverse it, contact the shop owner who can update it directly via Supabase if needed. In practice, advance carefully: Done is the only status that sends an external notification.

Inquiries

When a customer sends a product inquiry from the /#/shop page, it shows up in the Inquiries panel. A red badge on the dashboard tile tells you there are unread inquiries.

1

Open Inquiries from the dashboard

Tap the Inquiries tile. The table shows all inquiries with their status: New inquiries are bold with a yellow left border — these need attention first.

2

Click to see the full inquiry

Click any row to expand it. You'll see the customer's full item list with quantities and prices, their message to the shop, and their contact details (phone and/or email).

3

Contact the customer directly

Use the phone number or email shown in the expanded row to reach out. The app doesn't send replies — you contact them yourself, outside the system.

4

Update the status

Once you've replied: tap Mark Replied. If the inquiry is resolved or doesn't need a response: tap Mark Closed. The dashboard badge count drops.

New inquiry badge The badge on the dashboard tile only counts New inquiries. Once you mark an inquiry Replied or Closed, it stops counting toward the badge — even if it's still in the table.

Quick Reference

Order statuses

  • Pending Not started yet
  • In Progress Being strung now
  • Done Ready — emails customer
  • Picked Up Customer collected it

Inquiry statuses

  • New Needs a response
  • Replied You've contacted them
  • Closed No further action

Key actions

  • Tap status badge → cycle status forward
  • Click row → expand details
  • Mark Replied → clears New badge

Who does what

  • Customer → submits kiosk order
  • Staff → updates queue
  • Owner → manages products & settings
